01

App Structure

Every Vue 3 app follows this skeleton. The five core options live inside createApp({}).

data()
Returns an object of reactive variables. Any change here auto-updates the DOM.
methods
Functions you call from the template. Use this.x to access data.
computed
Derived values. Re-run only when their dependencies change (cached).
watch
Runs a function whenever a specific data property changes.
created
Lifecycle hook — runs once when the app instance is first created.
// ① Load Vue from CDN
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

// ② Create & mount the app
const { createApp } = Vue

createApp({
  data() {
    return {
      yourName: '',      // reactive variable
    }
  },

  methods: {           // functions
    greet() { return 'Hi, ' + this.yourName }
  },

  computed: {          // derived / cached values
    shout() { return this.yourName.toUpperCase() }
  },

  watch: {             // side-effects on change
    yourName(newV, oldV) { console.log(newV) }
  },

  created() {          // runs once on startup
    console.log('App ready!')
  }

}).mount('#app')  // ③ attach to <div id="app">
MVVM: The HTML is your View, data() is your Model, and Vue is the ViewModel binding them together automatically.
02

v-model — Two-way Binding

Links an <input> to a data property — when the user types, the variable updates instantly, and vice versa.

v-model="varName" v-model.lazy="varName" works on: input, textarea, select, radio, checkbox 
<div id="app">
  <!-- Text input bound to 'yourName' -->
  <input v-model="yourName" type="text" />

  <!-- .lazy only updates when the field loses focus -->
  <input v-model.lazy="fruit" type="text" />

  <!-- Radio buttons share the same v-model variable -->
  <input type="radio" v-model="subject" value="Math"> Math
  <input type="radio" v-model="subject" value="English"> English

  <!-- {{ }} interpolation displays the value -->
  <p>Hello, {{ yourName }}!</p>
</div>

<script>
createApp({
  data() {
    return { yourName: '', subject: '', fruit: 'apple' }
  }
}).mount('#app')
</script>
03

v-bind — Dynamic Attributes

Binds an HTML attribute to a data value. Shorthand: replace v-bind: with just :

v-bind:href="url" :src="imageSrc" :class="myClass" 
<div id="app">
  <!-- Long form -->
  <a v-bind:href="someUrl">Visit Swinburne</a>

  <!-- Shorthand (: prefix) — same result -->
  <img :src="imageSrc" alt="flower">
</div>

<script>
createApp({
  data() {
    return {
      someUrl: 'https://swin.edu.au',
      imageSrc: 'flower.jpg'
    }
  }
}).mount('#app')
</script>
Key distinction: Use {{ }} for text content inside a tag. Use v-bind: / : for HTML attributes.
04

v-if / v-else — Conditional Rendering

Shows or hides elements based on a condition. The element is removed from the DOM entirely when false (unlike v-show which just hides it with CSS).

<div id="app">
  <!-- Buttons to set the reactive variable -->
  <button v-on:click="animal='cat'">Cats</button>
  <button v-on:click="animal='dog'">Dogs</button>

  <!-- Only renders when animal == 'cat' -->
  <div v-if="animal == 'cat'">🐱 Cat section</div>

  <!-- Renders for everything else -->
  <div v-else>🐶 Dog section</div>
</div>

<script>
createApp({
  data() { return { animal: 'cat' } }
}).mount('#app')
</script>
05

v-for — List Rendering

Loops over an array and renders an element for each item. Works on any HTML element or component.

v-for="item in array" v-for="(item, index) in array" v-for="obj in objectArray" 
<div id="app">
  <!-- Simple array of strings -->
  <ul>
    <li v-for="s in students">{{ s }}</li>
  </ul>

  <!-- Array of objects — access properties with dot notation -->
  <tr v-for="m in filterMarks">
    <td>{{ m.sName }}</td>
    <td>{{ m.subject }}</td>
    <td>{{ m.mark }}</td>
  </tr>
</div>

<script>
createApp({
  data() {
    return {
      students: ['Amy', 'Bill', 'Casey'],
      filterMarks: [
        { sName: 'Amy', subject: 'Math', mark: 88 },
        { sName: 'Bill', subject: 'English', mark: 74 },
      ]
    }
  }
}).mount('#app')
</script>
06

v-on — Event Handling

Listens for DOM events (click, dblclick, input…) and runs an expression or method. Shorthand: replace v-on: with @

v-on:click="handler" @click="handler" v-on:dblclick="..." @click="val = 'something'" 
<div id="app">
  <!-- Inline expression — no method needed for simple changes -->
  <button v-on:click="count++">Click me ({{ count }})</button>

  <!-- Shorthand @ — calls a method -->
  <button @click="changeText">Change Text</button>

  <!-- Single vs double click -->
  <button v-on:click="clk='Single'" v-on:dblclick="clk='Double'">
    Click or Double-click
  </button>
  <p>{{ clk }}</p>

  <!-- Toggle a boolean -->
  <button @click="show = !show">Toggle</button>
  <h2 v-if="show">Visible!</h2>
</div>
07

Methods — Functions

Methods are plain functions. Call them from the template with (). Always use this.x to access data properties inside a method.

<div id="app">
  <input v-model="fname">
  <input v-model="time">

  <!-- Call method with arguments directly in template -->
  <p>{{ sayhello(fname, time) }}</p>

  <!-- Or call a no-argument method on button click -->
  <button @click="checkGuess()">Check</button>
</div>

<script>
createApp({
  data() {
    return { fname: 'Amy', time: '3', guess: '', secret: 0, msg: '' }
  },
  methods: {
    // Method with parameters
    sayhello(fname, time) {
      if (time < 12)       return 'Good morning, ' + this.fname
      else if (time < 18)  return 'Good afternoon, ' + this.fname
      else                  return 'Good evening, ' + this.fname
    },

    // Method accessing data with 'this'
    checkGuess() {
      if (this.secret % 2 === 0 && this.guess === 'even')
        this.msg = 'Correct!'
      else
        this.msg = 'Wrong!'
    },

    // Helper method called by other methods
    genRandomNum() {
      return Math.floor(Math.random() * 100) + 1
    }
  },

  created() {   // auto-run on startup
    this.secret = this.genRandomNum()
  }
}).mount('#app')
</script>
08

Computed — Cached Derived Values

Like a method, but cached — only re-calculates when its reactive dependencies change. Use it for derived display logic instead of methods when the result depends purely on data.

✅ Use computed when

The value is derived from data and doesn't need arguments. e.g. reversing a string, filtering a list.

✅ Use methods when

You need to pass arguments, trigger a side effect, or call it from an event handler.

<div id="app">
  <input v-model="msg" placeholder="Type something...">

  <!-- Computed property used like a data variable (no parentheses!) -->
  <p>Reversed: {{ reversedMessage }}</p>
</div>

<script>
createApp({
  data() { return { msg: '' } },

  computed: {
    reversedMessage() {
      // Only re-runs when this.msg changes
      return this.msg.split('').reverse().join('')
    },

    // Practical example: filter a list based on user input
    filterMarks() {
      return this.marks.filter(m =>
        m.sName.includes(this.stuObj.sName) &&
        m.subject.includes(this.stuObj.subject)
      )
    }
  }
}).mount('#app')
</script>
09

Watch — Side Effects on Change

Runs a function whenever a specific data property changes. Good for async operations, alerts, or updating other data in response to a change.

<div id="app">
  <!-- .lazy = only trigger watch when field loses focus -->
  <input v-model.lazy="fruit">
  <h2>{{ msg }}</h2>
</div>

<script>
createApp({
  data() {
    return { fruit: 'apple', fName: '', lName: '', msg: '' }
  },
  watch: {
    // Receives (newValue, oldValue)
    fruit(newV, oldV) {
      this.msg = `Changed from ${oldV} to ${newV}`
    },

    // Watch multiple properties separately
    fName(newV, oldV) {
      alert(`First name: ${oldV} → ${newV}`)
    },
    lName(newV, oldV) {
      alert(`Last name: ${oldV} → ${newV}`)
    }
  }
}).mount('#app')
</script>
Tip: Use v-model.lazy with watch so the watcher only fires when the user finishes typing and moves focus away — not on every keypress.
10

Components — Reusable Pieces

Components are mini-apps you can reuse. Each has its own data() (so state is independent) and a template. Pass data in from the parent with props.

<div id="app">
  <!-- Each instance has its own counter -->
  <data-function-example></data-function-example>
  <data-function-example></data-function-example>

  <!-- Pass a prop using v-bind -->
  <blog-post title="My Vue Journey"></blog-post>
  <my-component :fruits="['apple','banana']"></my-component>
</div>

<script>
const app = createApp({})

// Component with independent state (data as function = each instance isolated)
app.component('data-function-example', {
  data: function() { return { counter: 0 } },
  template: `<button @click="counter++">Count: {{ counter }}</button>`
})

// Component that receives a string prop
app.component('blog-post', {
  props: ['title'],
  template: '<p>{{ title }}</p>'
})

// Component that receives an array prop, renders with v-for
app.component('my-component', {
  props: ['fruits'],
  template: '<p v-for="f in fruits">{{ f }}</p>'
})

app.mount('#app')
</script>
11

Custom Directives

Build your own v-something directives when you need to directly manipulate DOM elements. The el argument is the raw DOM element.

<div id="app">
  <!-- No value — hardcoded in directive -->
  <p v-highlight>Light green background</p>

  <!-- Pass a string value -->
  <p v-highlight1="'pink'">Pink background</p>

  <!-- Pass an object with multiple properties -->
  <p v-highlight2="{color:'lightblue', border:'navy 3px solid'}">
    Blue + border
  </p>
</div>

<script>
const app = createApp({})

// el = the DOM element, binding.value = what you passed in
app.directive('highlight', {
  created(el) { el.style.backgroundColor = 'lightgreen' }
})

app.directive('highlight1', {
  created(el, binding) { el.style.backgroundColor = binding.value }
})

app.directive('highlight2', {
  created(el, binding) {
    el.style.backgroundColor = binding.value.color
    el.style.border          = binding.value.border
    el.style.fontStyle       = binding.value.font
  }
})

app.mount('#app')
</script>

Vue Router

Enables client-side navigation between "pages" without a full page reload. Requires the vue-router library in addition to Vue.

<!-- ① Load both libraries -->
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
<script src="https://unpkg.com/vue-router@4"></script>

<!-- ② Navigation links (replaces <a href>) -->
<router-link to="/">Home</router-link>
<router-link to="/about">About</router-link>

<!-- ③ Where matched component renders -->
<router-view></router-view>

<script>
// Step 1 — Define components for each route
const Home  = { template: '<div><h2>Home</h2></div>' }
const About = { template: '<div><h2>About</h2></div>' }

// Step 2 — Map URL paths to components
const routes = [
  { path: '/',       component: Home },
  { path: '/about',  component: About },
]

// Step 3 — Create the router instance
const router = VueRouter.createRouter({
  history: VueRouter.createWebHashHistory(),  // uses # in URL
  routes
})

// Step 4 — Mount app with router plugin
const app = Vue.createApp({})
app.use(router)
app.mount('#app')
</script>

<router-link>

Replaces <a href> for navigation. Prevents full page reload and updates the URL.

<router-view>

Acts as a placeholder — Vue swaps in the matching component here when the route changes.
