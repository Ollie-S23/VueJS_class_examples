01

HTML Built-in Validation

The browser can handle basic validation with nothing but HTML attributes — no JavaScript needed. Good for quick constraints, but limited in what error messages you can show.

required
Field must not be empty before submitting.
minlength / maxlength
Text must be within the character count range.
min / max
Number or date must be within the given range.
type="email"
Browser checks for a valid email format automatically.
type="number"
Only allows numeric input.
type="date"
Renders a native date picker.
pattern="..."
Input must match the given regular expression. e.g. \d{4} = exactly 4 digits.
<form method="post" action="formtest.php">

  <!-- required + length limits -->
  <input type="text"   name="fName"
         required minlength="5" maxlength="8">

  <!-- numeric range -->
  <input type="number" name="age"
         min="10" max="50">

  <!-- date picker -->
  <input type="date" name="date">

  <!-- regex pattern: exactly 4 digits -->
  <input type="text" name="postcode"
         pattern="\d{4}">

  <input type="submit" value="Submit">
</form>
02

Vue Form Validation

For custom error messages and more control, validate inside a Vue method. Hook into the form's @submit event and prevent submission if validation fails.

<!-- HTML: attach checkForm to the submit event -->
<form method="post" @submit="checkForm"
      action="formtest.php">

  <!-- Error list — only shows when errors array is non-empty -->
  <div v-if="errors.length">
    <ul>
      <li v-for="e in errors">{{ e }}</li>
    </ul>
  </div>

  <input type="text" v-model="fName">
  <input type="text" v-model="lName">
  <input type="submit" value="Submit">
</form>
// simple form.js
createApp({
  data() {
    return {
      fName: '',
      lName: '',
      errors: []    // collects all error messages
    }
  },
  methods: {
    checkForm(e) {
      var result = true
      this.errors = []    // reset errors on each attempt

      if (!this.fName.trim()) {     // .trim() removes whitespace
        result = false
        this.errors.push('Please enter first name.')
      }
      if (!this.lName.trim()) {
        result = false
        this.errors.push('Please enter last name.')
      }

      if (!result)
        e.preventDefault()  // block form submission
    }
  }
}).mount('#app')
03

e.preventDefault() — Stopping Submission

When a form submits, the browser normally sends the data and navigates away. e.preventDefault() cancels that default behaviour so you can show errors instead.

Without preventDefault

Form submits → page navigates to action URL → errors can't be shown.

With preventDefault

Submission is blocked → user stays on page → error messages appear in the DOM.

// The 'e' is the submit Event object
checkForm(e) {
  var valid = true

  // ... run your checks, set valid = false if anything fails

  if (!valid) {
    e.preventDefault()   // cancel the browser's default submit
  }
  // If valid stays true, form submits normally
}
Note: The event object e is automatically passed to the method when using @submit="checkForm" — you don't need to do anything special to receive it.

Vuetify — Setup

Vuetify is a UI component library for Vue 3 built on Material Design. It gives you pre-styled, accessible components like buttons, text fields, alerts, and more with minimal CSS writing.

<!-- 1. Load Material Design Icons -->
<link href="https://cdn.jsdelivr.net/npm/@mdi/font@6.x/css/materialdesignicons.min.css"
      rel="stylesheet">

<!-- 2. Load Vuetify CSS -->
<link href="https://cdn.jsdelivr.net/npm/vuetify@3.5.14/dist/vuetify.min.css"
      rel="stylesheet">

<!-- 3. Load Vue, then Vuetify JS -->
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
<script src="https://cdn.jsdelivr.net/npm/vuetify@3.5.14/dist/vuetify.min.js"></script>

<script>
const { createApp }    = Vue
const { createVuetify } = Vuetify   // destructure from Vuetify global

const vuetify = createVuetify()    // create Vuetify instance
const app     = createApp({ /* your options */ })

app.use(vuetify)   // register Vuetify as a Vue plugin
app.mount('#app')  // mount as normal
</script>
Order matters: Always load Vue JS before Vuetify JS. And always call app.use(vuetify) before app.mount().

Vuetify Layout Components

Vuetify has a standard page structure. Most content goes inside <v-main>, and almost everything must be wrapped in <v-app>.

<v-app>
Root wrapper — required for Vuetify to work. Everything goes inside this.
<v-app-bar>
Fixed top navigation bar. Use app prop to lock it in place.
<v-main>
The main content area. Adjusts automatically when app-bar/footer are present.
<v-container>
Centers and pads content. Like Bootstrap's container. Add fluid to go full-width.
<v-footer>
Fixed bottom footer. Use app prop to lock it in place.
<div id="app">
  <v-app>                         <!-- required root -->

    <v-app-bar app>
      <p>My App Bar</p>
    </v-app-bar>

    <v-main>
      <v-container fluid>   <!-- fluid = full width -->

        <!-- Vuetify button with MDI icon -->
        <v-btn color="primary">
          <v-icon>mdi-pencil</v-icon>
        </v-btn>
        <v-btn color="error">
          <v-icon>mdi-alarm</v-icon>
        </v-btn>

        <!-- Vuetify alerts -->
        <v-alert type="success">Success message</v-alert>
        <v-alert type="info">Info message</v-alert>

      </v-container>
    </v-main>

    <v-footer app><p>Footer</p></v-footer>

  </v-app>
</div>
color="primary" color="error" color="success" color="info" type="warning"

Vuetify Form Components

Vuetify's form components look polished by default and integrate with a rules prop for inline validation — no manual error list needed.

<v-form method="post" action="formtest.php">

  <!-- Text field — label floats above when focused -->
  <v-text-field
    name="firstName"
    v-model="firstName"
    :rules="nameRules"     <!-- array of validation functions -->
    label="First Name"
    required
  ></v-text-field>

  <!-- Email field -->
  <v-text-field
    name="email"
    v-model="email"
    type="email"
    :rules="emailRules"
    label="Email"
  ></v-text-field>

  <v-btn type="submit" color="success">Submit</v-btn>

</v-form>
Key difference from plain HTML: <v-text-field> renders a full Material Design input with floating label, underline, and inline error display — automatically driven by the :rules array.
07

The :rules Array

Rules are an array of arrow functions — each receives the current field value v and must return either true (valid) or an error string (invalid). Vuetify shows the string automatically under the field.

// Each rule: v => true  OR  v => 'error message'

nameRules: [

  // Rule 1: field must not be empty
  // !! converts a value to boolean — empty string becomes false
  v => !!v || 'Name is required',

  // Rule 2: max length
  v => (v && v.length <= 10) || 'Name must be less than 10 characters',

  // Rule 3: letters only — using a regular expression
  v => /^[a-zA-Z]+$/.test(v) || 'Name can only contain letters'
],

emailRules: [
  v => !!v || 'E-mail is required',

  // Regex checks for a valid email pattern
  v => /[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,3}$/.test(v)
       || 'E-mail must be valid'
]

!! (double NOT)

Converts any value to a strict boolean. !!'' → false, !!'Amy' → true. Used to check if a field is non-empty.

/regex/.test(v)

Tests the value against a regular expression. Returns true if it matches, false if not. Used for format checks (email, letters only, postcodes).

Arrow function shorthand: v => !!v || 'msg' is the same as writing function(v) { return !!v || 'msg' }. The || means: if the left side is truthy (valid), return it; otherwise return the error string.

Vuetify Form Controls

Vuetify has a full set of form controls that all support v-model and the :rules prop. Here's the full set from your files:

<v-form method="post" action="formtest.php">

  <!-- TEXT FIELD — for single-line text / email -->
  <v-text-field name="email" v-model="email"
    type="email" :rules="emailRules" label="Email">
  </v-text-field>

  <!-- RADIO GROUP — one choice only -->
  <v-radio-group name="sex" v-model="sex">
    <v-radio label="Male"   value="M" color="red"></v-radio>
    <v-radio label="Female" value="F" color="green"></v-radio>
    <v-radio label="Other"  value="O" color="blue"></v-radio>
  </v-radio-group>

  <!-- CHECKBOXES — multiple choices; v-model binds to an array -->
  <v-layout>
    <v-checkbox name="prog[]" v-model="program"
      label="P01 Mathematics" value="P01"></v-checkbox>
    <v-checkbox name="prog[]" v-model="program"
      label="P02 Swimming"    value="P02"></v-checkbox>
  </v-layout>

  <!-- SELECT DROPDOWN — bound to a list of options -->
  <v-select name="state" v-model="state"
    :items="stateList" label="State">
  </v-select>

  <!-- TEXTAREA — for multi-line text -->
  <v-textarea name="comments" label="Comments"
    v-model="comments">
  </v-textarea>

  <v-btn type="submit" color="success">Submit</v-btn>
</v-form>
// vuetify form controls.js — data setup
createApp({
  // Arrow function shorthand for data() — same as data() { return {...} }
  data: () => ({
    firstName: '',
    email:     '',
    sex:       'F',            // radio default
    program:   ['P02'],         // checkboxes bind to an ARRAY
    stateList: ['VIC','NSW','WA'], // options for v-select
    state:     'VIC',
    comments:  '',
    nameRules:  [ /* ... */ ],
    emailRules: [ /* ... */ ]
  })
}).use(vuetify).mount('#app')

Checkbox — array binding

Multiple checkboxes sharing the same v-model name all bind to one array. Each checked box adds its value to the array. Pre-check one by including its value in the default: program: ['P02']

v-select — items prop

Pass your list of options via :items="stateList" where stateList is an array in data(). The selected value is stored in the bound variable (state).
